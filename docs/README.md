## 1. 项目目标
这是一个面向中国企业赴迪拜参展场景的翻译服务撮合与履约 App。

## 2. 项目文档入口
AI 或开发者在开始任何开发前，必须先阅读以下文件：
1. `PRD.md`
2. `architecture.md`
3. `api-contract.md`
4. `README.md`

## 3. 开发原则
- 不允许跳过文档直接写代码
- 不允许新增未定义字段
- 不允许新增未定义状态
- 不允许擅自修改角色权限
- 前后端必须共享同一套枚举定义
- 所有金额统一 AED
- 所有展会示例默认使用 Dubai / Dubai World Trade Centre
- 不得为了兼容旧前端而修改后端字段命名
- 页面字段、provider、service、model 必须与 `api-contract.md` 一致

补充：
- 对你这种以 AI 为主要开发执行者的项目，关键业务说明不能为了“文档简洁”而删除。
- 字段来源说明、脱敏规则说明、状态语义说明、接口副作用说明，都属于必须长期保留的内容。

## 5. 推荐开发顺序
1. 阅读 docs 下 4 个文件
2. 建 shared 枚举与 schema
3. 建数据库表结构
4. 建后端 REST API
5. 建前端页面壳子
6. 用 mock 数据跑通页面
7. 切真实 API
8. 联调与验收
9. 每轮完成后更新文档

当前阶段补充顺序：
1. 先保证工程可编译、可运行
2. 再保证主链路可联调
3. 再补齐最小 MVP 验证闭环
4. 最后再扩非主链路功能

## 6. 每轮开发完成后必须更新什么
### 6.1 如果新增业务字段
更新：
- `PRD.md`
- `api-contract.md`
- 如影响实体，再更新 `architecture.md`

### 6.2 如果新增状态或流程
更新：
- `architecture.md`
- `PRD.md`
- `api-contract.md`

### 6.3 如果只是改实现方式
更新：
- `README.md`
- 必要时更新 `architecture.md`

### 6.4 如果只是增加验证闭环的开发环境入口
更新：
- `PRD.md`
- `architecture.md`
- `api-contract.md`
- `README.md`

说明：
- 例如“开发环境模拟审核通过”这类能力虽然不是正式生产业务，但会影响流程验证与实现边界，仍必须文档化。

## 7. AI 执行前固定指令
可将下面这段放到你给 AI 的每次任务前：

```text
在执行任何代码修改前，请先完整阅读 docs/PRD.md、docs/architecture.md、docs/api-contract.md、docs/README.md。
你必须把这四个文件视为当前项目的唯一业务依据。
如果你的实现与文档冲突，优先修改方案，不要直接修改代码。
如果你认为需要新增字段、状态、接口或流程，请先提出文档变更，再执行代码。
完成任务后，请输出：
1. 本次改动内容
2. 是否影响 PRD / architecture / api-contract / README
3. 如有影响，请给出应更新的具体段落
```

## 8. 文档维护规则
### 8.1 版本记录建议
每次更新文档，在文件顶部追加：
- 更新时间
- 修改人
- 修改原因

### 8.2 修改策略
- 小改动：直接增量改文档
- 大改动：先列“变更摘要”，再改正文
- 不允许代码先于文档长期漂移

### 8.3 本次修订策略
本次文档修订遵循：
- 保留原有关键说明
- 合并新增 MVP 内容
- 不为了压缩文档而删除解释性规则

## 9. 当前开发阶段
当前项目处于”MVP 主链路 + 辅助功能基本完成”的阶段。

本阶段已完成：
- 登录 / 注册 / 身份选择
- 雇主提交需求
- 翻译员查看机会并提交报价
- 雇主确认报价生成订单
- 翻译员执行订单履约动作
- 订单列表与订单详情展示
- 开发环境模拟审核通过
- 我的需求列表与需求详情到订单追踪
- 收藏翻译员、收藏列表
- 档期管理（日历视图 + 批量设置）
- 雇主评价翻译员
- 翻译员工作台统计（真实数据）
- 最小聊天系统（文本消息 + 需求卡片 + 图片消息，双方均可发图）
- 雇主导航重构：[首页, 找翻译, 聊天, 订单, 我的]
- 翻译员详情页按钮：[收藏, 立即沟通, 邀请报价（自动聊天邀请，已关闭需求不可邀请）]
- 需求详情页翻译员私聊入口
- 翻译员导航重构：[工作台, 档期, 聊天, 订单, 我的]
- 翻译员工作台快捷入口：[查看新需求, 报价, 完善资料]
- 需求审核状态（PENDING_REVIEW → APPROVED/REJECTED），翻译员仅可见已审核通过需求
- 站内通知系统（通知列表、未读计数红点、点击跳转关联实体、全部已读）
- 聊天需求卡片点击跳转详情页（翻译员 → 机会详情，雇主 → 需求详情）
- 雇主/翻译员首页通知铃铛 + 未读红点

本阶段暂不扩展：
- common/enums
- upload
- 支付、收入结算
- 实时消息推送（WebSocket）
- marketplace 高级筛选与完整联调

## 10. 你的最佳工作流
对你这种主要依赖 AI 开发的人，最稳的方式是：
- 先用文档约束 AI
- 再让 AI 写代码
- 每完成一个模块就让 AI 回写文档
- 你主要在页面与接口层面验收，不依赖 terminal 做全部验收

补充建议：
- 非主链路页面可以暂时降级，但整个工程必须始终可编译、可运行
- 当前阶段优先追求“你能肉眼看到并走完闭环”，而不是一次性做完整目标

- 这会显著减少 AI 重复犯错
