## 1. 产品概述
### 1.1 产品名称
展会翻译 App

### 1.2 产品定位
面向中国企业赴迪拜参展场景的翻译服务撮合与履约平台。平台不是泛翻译平台，只服务明确的展会翻译业务。

### 1.3 核心身份
同一个 App 内包含两类身份：
- 雇主：发布翻译需求、筛选译员、收藏、提交需求、跟踪订单
- 翻译员：完善资料、管理档期、查看匹配需求、提交报价、跟踪订单、处理履约

### 1.4 MVP 范围
包含：
- 登录 / 注册
- 身份选择
- 雇主找翻译、收藏、提交需求、订单跟踪
- 翻译员入驻、审核状态、档期、需求报价、订单履约
- 统一订单状态流转

不包含：
- 在线支付
- 社交内容流
- 多国家 / 多城市泛化运营后台
- 复杂结算系统

已包含（轻量版）：
- 最小聊天系统（文本消息 + 需求卡片 + 图片消息，无实时推送、无已读回执）
- 站内通知系统（平台通知列表、未读计数、点击跳转关联实体）

### 1.5 当前阶段范围
当前主链路已联调范围：
- 登录 / 注册 / 身份选择
- 雇主提交需求
- 翻译员查看机会列表与机会详情
- 翻译员提交报价
- 雇主确认报价生成订单
- 雇主订单列表 / 订单详情
- 翻译员订单列表 / 订单详情 / 履约动作

当前为补齐 MVP 验证闭环需新增或补强的最小能力：
- 翻译员资料提交后的“审核通过”验证入口
- 雇主“我的需求”列表
- 雇主需求详情中的报价追踪与订单入口

当前未纳入主链路联调：
- 找翻译列表与详情
- 收藏
- 工作台完整数据
- 档期管理
- Common / Upload / Support 扩展接口

## 2. 产品目标
### 2.1 业务目标
- 帮助雇主快速提交迪拜展会翻译需求
- 帮助翻译员接收匹配需求并提交报价
- 帮助平台完成订单确认与履约跟踪

### 2.2 成功指标
- 雇主成功提交需求率
- 翻译员资料提交完成率
- 需求到报价转化率
- 报价到订单确认转化率
- 已确认订单履约完成率

### 2.3 当前阶段验证目标
本阶段不是追求“大而全”，而是确保以下两条闭环都能通过 UI 验证：
1. 翻译员：注册 / 登录 -> 完善资料 -> 审核通过 -> 查看需求 -> 提交报价
2. 雇主：提交需求 -> 查看我的需求 -> 查看报价 -> 确认报价 -> 生成订单 -> 跟踪订单进度

## 3. 用户与角色定义
### 3.1 雇主
典型用户：赴迪拜参展的中国企业工作人员

可执行操作：
- 浏览首页、找翻译、详情页
- 收藏翻译员
- 提交翻译需求
- 查看自己发布的需求及其进度
- 查看订单列表与详情
- 管理个人信息、公司信息、开票信息

限制：
- 未登录可浏览，但收藏、提交需求需先登录

### 3.2 翻译员
典型用户：服务迪拜展会场景的本地或驻场翻译员

可执行操作：
- 登录并选择“我是翻译员”
- 提交入驻资料
- 管理档期
- 查看新需求与需求详情
- 提交报价
- 查看订单与履约状态

限制：
- 未通过审核只能完善资料、查看审核状态、联系客服
- 未通过审核不能报价、不能接单、不能进入主链路机会列表

## 4. 关键业务规则
### 4.1 身份规则
- 这是同一个 App，不拆分为两个独立 App
- 首次登录必须选择身份：我是雇主 / 我是翻译员
- 当前 MVP 不支持一个账号同时激活两套并行业务身份；如需切换，需在账户设置中申请切换或新增身份能力（后续版本）

### 4.2 订单状态统一枚举
所有订单统一使用以下状态：
- 待报价
- 待确认
- 已确认
- 服务中
- 已完成
- 已取消

说明：
- 当前正式创建后的订单从“待确认”开始。
- “待报价”仍用于产品层状态统一与筛选语义。

### 4.3 联系方式展示规则
- 订单确认前，客户联系方式弱化或脱敏
- 订单确认后，完整展示联系人与到场信息
- 雇主侧始终可查看自己提交的完整联系人信息

### 4.4 报价规则
- 翻译员可按小时 / 按天 / 按项目报价
- 报价需包含金额、服务时段、说明、税务说明
- 同一需求可存在多个翻译员报价，由雇主在待确认阶段选择
- 同一译员对同一需求不可重复报价

### 4.5 档期规则
- 档期状态：可接单 / 已占用 / 待确认 / 休息
- 已确认订单对应时间段不可再次接单
- 待确认时间段需明显提示，但允许平台后续做人工调整

### 4.6 需求审核规则
- 雇主提交需求后，需求进入 `PENDING_REVIEW`（待审核）状态
- 平台审核通过后进入 `APPROVED`，审核不通过进入 `REJECTED`
- 仅审核通过（`APPROVED`）的需求对翻译员可见
- 雇主侧"我的需求列表"展示审核状态标签（审核中 / 审核未通过）
- 当前阶段提供开发环境"模拟审核通过"入口

### 4.7 站内通知规则
- 平台在关键业务节点自动生成站内通知
- 通知事件包括：需求已提交、审核通过、审核拒绝、收到新报价、报价已确认、订单状态变更
- 通知列表按时间倒序展示，支持标记已读、全部已读
- 通知关联业务实体（需求或订单），点击可跳转到对应详情页
- 未读通知在首页/工作台以红点+数字展示

### 4.8 MVP 验证规则
- 当前阶段允许提供“开发环境模拟审核通过”能力，仅用于验证审核前后权限变化
- 当前阶段不做正式平台后台，也不要求通过 terminal 或手改数据库才能完成核心流程验证
- 雇主提交需求后，必须能从客户端看到自己发布的需求，而不是只能提交不能追踪
- 需求页负责看报价与确认报价，订单页负责履约追踪，两者不能混淆

- 评价摘要

- 已关闭（CLOSED）或已取消（CANCELLED）的需求不可用于邀请报价，需求选择器中自动过滤。

核心字段：

### 5.2.7 我的需求列表页
目标：让雇主查看自己发布的需求，并追踪需求当前进度

功能：
- 列表展示自己提交的所有需求
- 支持按状态查看：全部 / 待报价 / 报价中 / 已生成订单
- 卡片显示：展会名称、城市、展馆、日期范围、当前 `requestStatus`、报价数、是否已生成订单
- 支持进入需求详情
- 若已生成订单，提供“查看订单进度”入口

说明：
- 该页面是本阶段新增的 MVP 验证页面
- 目的是解决“提交了需求，但从客户端看不到需求和进度”的断链问题

### 5.2.8 需求详情页
功能：
- 展示展会信息、语言需求、预算信息、联系人信息、备注
- 展示当前 `requestStatus`
- 展示已收到的报价列表
- 支持雇主确认某条报价
- 若已生成订单，显示“查看订单进度”入口

说明：
- 看报价、确认报价发生在需求详情页，而不是订单生成前直接在订单列表中完成
- 确认报价后，此需求进入 `CLOSED`，并生成订单

### 5.2.9 订单列表页
功能：
- 按状态查看：全部 / 待报价 / 待确认 / 已确认 / 服务中 / 已完成 / 已取消
- 卡片显示：订单编号、展会、日期、翻译摘要、状态

### 5.2.10 订单详情页
功能：
- 订单信息
- 报价摘要
- 翻译信息
- 开票需求
- 状态时间线
- 根据状态展示不同 CTA

补充：
- 雇主在需求详情页即可确认报价，确认成功后进入订单详情。
- 订单详情页应能让用户理解其来源需求，可通过“来源需求”摘要或返回入口体现。

### 5.2.11 聊天列表页
功能：
- 展示当前用户的所有会话列表
- 客服会话始终置顶
- 每个会话显示：对方名称、最后一条消息、未读数、时间
- 点击进入聊天详情页

### 5.2.12 聊天详情页
功能：
- 展示与对方的消息列表
- 支持发送文本消息
- 雇主可发送需求卡片（从已发布的需求中选择）
- 双方均可上传并发送图片（单张，最大 10MB）
- 需求卡片展示：展会名称、日期、语言、地点、预算；点击卡片可跳转到需求详情（翻译员跳转机会详情，雇主跳转需求详情）
- 图片消息：缩略图展示，点击查看大图，加载失败有占位提示

### 5.2.13 我的页面
功能：
- 用户信息
- 公司信息
- 开票信息入口
- 我的需求（跳转到需求列表页）
- 我的收藏
- 我的订单
- 帮助与客服
- 退出登录

### 5.2.14 雇主导航栏
- Tab 顺序：首页 / 找翻译 / 聊天 / 订单 / 我的
- "需求"不再作为底部 tab，改为从首页和"我的"页面以 push 方式进入

## 6. 核心流程
### 6.1 雇主主流程
登录/注册 -> 选择“我是雇主” -> 浏览/筛选翻译员（后续阶段）或直接提交需求 -> 在“我的需求”中查看自己发布的需求 -> 在需求详情页查看报价 -> 确认报价 -> 订单创建 -> 查看订单详情 -> 服务进行 -> 服务完成

### 6.2 翻译员主流程
登录/注册 -> 选择“我是翻译员” -> 提交入驻资料 -> 审核通过 -> 查看新需求 -> 查看需求详情 -> 提交报价 -> 订单确认 -> 确认档期 -> 确认到场 -> 开始服务 -> 完成服务

### 6.3 当前阶段最小验证闭环
闭环 A（翻译员）：
- 注册 / 登录
- 完善资料
- 审核通过
- 查看新需求
- 提交报价

闭环 B（雇主）：
- 提交需求
- 查看我的需求
- 查看需求详情与报价
- 确认报价
- 生成订单
- 跟踪订单进度
